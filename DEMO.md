# Forge AI

## Official Demo Script

Version: 1.0

---

# Goal

The demo must convince judges that Forge AI understands an entire software system rather than individual files.

Everything shown in the demo must be real.

Never fake functionality.

Never hardcode responses.

---

# Demo Duration

Target

2 minutes 30 seconds

Maximum

3 minutes

---

# Demo Repository

Use a real-world repository.

Preferred options

1. FastAPI
2. Flask
3. Django
4. React
5. Next.js
6. LangChain
7. Kubernetes (subset)
8. VS Code Extension Sample

Repository should contain

100–1000 source files.

Avoid extremely small demo projects.

---

# Demo Flow

## Scene 1

Opening (10 seconds)

Open Forge AI.

Professional dashboard.

Dark theme.

Repository already indexed.

Display:

- Repository Name
- Languages
- Files
- Classes
- Functions
- Dependencies
- Architecture Health Score

Narration

"Forge AI builds a complete architectural understanding of an existing software repository."

---

## Scene 2

Architecture Visualization (25 seconds)

Open Architecture Graph.

Show:

Zoom

Pan

Collapse

Expand

Search

Highlight dependency path

Click a service.

Display:

Dependencies

Methods

Classes

Documentation

Narration

"Instead of reading hundreds of files, developers can navigate the entire architecture visually."

---

## Scene 3

Repository Intelligence (35 seconds)

Open AI Chat.

Ask

Explain how authentication works.

Expected

Repository-aware explanation.

Supporting files.

Supporting classes.

Dependency chain.

Confidence score.

Then ask

What happens if I modify auth.py?

Expected

Impact analysis.

Affected modules.

Potential risks.

---

## Scene 4

Bug Investigation (30 seconds)

Paste stack trace.

Forge AI should

Identify likely source.

Explain why.

Highlight related files.

Suggest fix.

Display confidence.

---

## Scene 5

Technical Debt (25 seconds)

Open dashboard.

Show

Complexity

Circular dependencies

Dead code

Large classes

Architecture violations

Risk score

Click one issue.

Show explanation.

---

## Scene 6

Documentation (20 seconds)

Generate

Architecture Documentation.

README.

Mermaid Diagram.

Display generated documentation.

---

## Scene 7

Search (15 seconds)

Search

PaymentService

Open result.

Jump directly to implementation.

Highlight dependencies.

---

## Scene 8

Closing (20 seconds)

Summarize

Repository

Architecture

Documentation

Bug analysis

Technical debt

AI reasoning

Narration

"Forge AI reduces software understanding from days to minutes by combining static analysis, graph reasoning, and GPT-5.6 into one developer workflow."

# Success Criteria

Judges should finish the demo believing:

1. The application understands software architecture.

2. GPT-5.6 is grounded in repository knowledge.

3. Codex was heavily used throughout development.

4. The product could realistically be used by software teams.
